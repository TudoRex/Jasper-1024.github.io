创建通知

  • 行文前：
    Android每次版本几乎都会有通知API的改动，不同版本之间通知兼容性很是问题，为此我们使用support_v7 库中的NotificationCompat.Builder代替Notification.Builder二者使用方式相同

  • 涉及到的两个类

    • NotificationManager
    • Notification (support库中对应NotificationCompat)

  • NotificationManager

    • 状态栏通知的管理类，负责发通知、清除通知等
    • NotificationManager 是一个系统Service，必须通过 getSystemService()方法来获取

  • Notification

    • 具体的状态栏通知对象，可以设置icon、文字、提示声音、振动等参数
    • 一个Builder对象至少包含三个方面
      • 一个小图标，通过setSmallIcon()方法设置。
      • 通知标题，通过setContentTitle()方法设置。
      • 详细文本，通过setContentText()方法设置。

PendingIntent

  • 可以简单理解为延迟的Intent

  • PendingIntent 生命周期与主程序无关。通过 PendingIntent 可以调用起三种组件

    • 组件 Activity Service Broadcast
    • 对应 getActivity() getService() getBroadcast()

  • getXXX()

    • PendingIntent getxxx (Context context, 
      int requestCode, 
Intent intent, 
int flags)
    • Context context 上下文
    • int requestCode 请求码 一般传入0即可
    • Intent intent Intent 对象
    • int flags PendingIntent行为 有5种取值
      • FLAG_CANCEL_CURRENT 如果 PendingIntent 已经存在，则用新的PendingIntent 取代
      • FLAGIMMUTABLE PendingIntent不可变
      • FLAG_NO_CREATE PendingIntent如果已存在则不创建新实例
      • FLAG_ONE_SHOT PendingIntent只使用一次
      • FLAG_UPDATE_CURRENT PendingIntent已存在时，Intent 有了改动则更新 PendingIntent
